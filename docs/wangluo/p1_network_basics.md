# 网络世界的第一步：从零认识计算机网络

你每天都在用网络：刷手机、看视频、打游戏、发消息……但你有没有想过，当你在浏览器里输入一个网址并按下回车后，**到底发生了什么**？

本篇文档不需要你有任何网络基础。我们会从你最熟悉的日常生活经验出发，一步步揭开计算机网络的面纱。

::: tip 阅读建议
不要试图一次记住所有概念。先通读一遍，建立一个模糊的印象，后续在实践中你会不断回来翻阅，每次都会有新的理解。
:::

# 一、什么是互联网？

简单来说，**互联网（Internet）** 就是把全世界的电脑、手机、服务器等设备连接在一起的巨型网络。

> **类比：快递网络**
> 
> 想象一下中国的快递系统：每个城市有快递站（服务器），每个人有一个收货地址（IP 地址），快递员按照地址把包裹（数据）送到你手上。互联网的工作方式和这几乎一模一样——只不过"包裹"变成了网页、视频、消息等数据。

互联网的核心思想就两个：
- **连接**：让任意两台设备能够互相找到对方。
- **传输**：让数据能够可靠地从一台设备到达另一台设备。

# 二、网络是怎么连接的？

## 2.1 你的家庭网络

你家里很可能有这样一个设备——**路由器**（就是那个有天线、闪着小灯的盒子）。它是你家所有设备通往互联网的"大门"。

```
                        ┌──────────────┐
                        │   互联网      │
                        │  (Internet)   │
                        └──────┬───────┘
                               │ 光纤/网线
                        ┌──────┴───────┐
                        │   光猫/调制   │
                        │   解调器      │
                        └──────┬───────┘
                               │ 网线
                        ┌──────┴───────┐
                        │    路由器     │
                        │   (Router)    │
                        └──┬───┬───┬───┘
                     WiFi │   │   │ WiFi / 网线
                          │   │   │
                      ┌───┘   │   └───┐
                      ▼       ▼       ▼
                   手机     笔记本    台式机
```

- **光猫**：把光纤里的光信号转换成电信号（你的电脑能听懂的语言）。
- **路由器**：负责把你家里所有设备的网络请求汇总，通过一根线发出去，再把回来的数据分发给对应的设备。
- **WiFi**：无线连接方式，不用网线也能上网。本质上和网线一样，只是用电磁波代替了铜线里的电信号。

## 2.2 连接方式对比

| 连接方式 | 优点 | 缺点 | 适用场景 |
|---------|------|------|---------|
| **有线（网线）** | 稳定、速度快、延迟低 | 需要布线，不够灵活 | 台式机、服务器 |
| **WiFi** | 方便、灵活 | 可能不稳定、受距离影响 | 手机、笔记本 |
| **移动数据（4G/5G）** | 随时随地 | 有流量限制、信号覆盖不均 | 外出使用 |

::: info 动手试试
打开你的手机，进入 **设置 → WiFi → 点击已连接的网络**，你会看到手机的 **IP 地址**（通常类似 `192.168.1.xxx`）。这就是你手机在家庭网络中的"临时门牌号"。
:::

# 三、IP 地址 — 设备的"门牌号"

## 3.1 什么是 IP 地址？

在网络世界中，每台设备都需要一个唯一的标识，这就是 **IP 地址**（Internet Protocol Address）。

> **类比：家庭住址**
> 
> 快递员要给你送快递，必须知道你的地址。同样，你的电脑要从百度服务器获取网页，就必须知道百度服务器的 IP 地址。

最常用的 IP 地址格式是 **IPv4**，由 4 个数字组成，用点号分隔：

```
192.168.1.1
```

每个数字的范围是 0~255，所以一个 IPv4 地址看起来像这样：`A.B.C.D`。

## 3.2 查看你的 IP 地址

::: code-group
```bash [Windows]
# 打开命令提示符（Win+R → 输入 cmd → 回车），输入：
ipconfig
```

```bash [Mac / Linux]
# 打开终端，输入：
ifconfig
# 或者更现代的命令：
ip addr
```
:::

运行后你会看到类似这样的输出：

```
以太网适配器:
   IPv4 地址 . . . . . . . . . . : 192.168.1.105
   子网掩码  . . . . . . . . . . : 255.255.255.0
   默认网关  . . . . . . . . . . : 192.168.1.1
```

- **IPv4 地址**：你电脑在局域网中的地址。
- **子网掩码**：后面会讲，先混个眼熟。
- **默认网关**：通常是你的路由器地址，你的电脑通过它访问互联网。

## 3.3 公网 IP vs 私网 IP

并不是所有 IP 地址都能在互联网上直接使用。IP 地址分为两类：

| 类型 | 说明 | 示例 |
|------|------|------|
| **公网 IP** | 全球唯一，可以直接在互联网上访问 | `220.181.38.148`（百度的服务器之一） |
| **私网 IP** | 只在局域网内使用，不能直接从外网访问 | `192.168.1.105`（你家电脑的地址） |

> **类比：公司内线电话 vs 公开电话号码**
> 
> 公司内部的分机号（如 8001、8002）只在公司内部能打通，这就是"私网 IP"。而公司的总机号码（如 0731-88888888）是公开的，外部也能拨打，这就是"公网 IP"。

私网 IP 有三段保留地址：
- `10.0.0.0` ~ `10.255.255.255`
- `172.16.0.0` ~ `172.31.255.255`
- `192.168.0.0` ~ `192.168.255.255` ← **你最常遇到的**

::: tip 关键概念
你家里的每台设备都有一个私网 IP（如 `192.168.1.105`），但对外它们共用路由器的一个公网 IP。路由器负责在两者之间"翻译"，这个机制叫做 **NAT**（网络地址转换），我们在进阶篇会详细讲。
:::

# 四、MAC 地址 — 设备的"身份证号"

## 4.1 什么是 MAC 地址？

如果说 IP 地址像"门牌号"（可以搬家用新的），那 **MAC 地址** 就像"身份证号"（出厂就固定了）。

MAC 地址是写在网卡（网络硬件）上的唯一标识，格式如下：

```
AA:BB:CC:DD:EE:FF
```

由 6 组两位十六进制数组成，共 12 个字符。

## 4.2 IP 地址 vs MAC 地址

| 对比项 | IP 地址 | MAC 地址 |
|--------|---------|----------|
| **类比** | 门牌号 | 身份证号 |
| **是否可变** | 可以变化（换网络就换 IP） | 出厂固定（通常不变） |
| **作用范围** | 跨网络通信 | 同一局域网内通信 |
| **在哪一层** | 网络层 | 链路层 |

> **为什么两个都需要？**
> 
> IP 地址帮助数据包找到"目标网络"（比如找到你家小区），MAC 地址帮助数据包在局域网内找到"具体设备"（比如找到你家 302 室）。两者配合，数据才能精准送达。

# 五、域名与 DNS — 网络世界的"电话簿"

## 5.1 为什么需要域名？

你平时上网时输入的是 `www.baidu.com`，而不是 `220.181.38.148`。这是因为人类更容易记住有意义的名字，而不是一串数字。

**域名（Domain Name）** 就是 IP 地址的"好记版本"。

> **类比：手机通讯录**
> 
> 你打电话时找的是"妈妈"这个名字，而不是背下 138xxxx1234 这串号码。DNS 就是互联网的"通讯录"。

## 5.2 域名的结构

以 `www.baidu.com` 为例：

```
www    .   baidu   .   com
 ↑          ↑          ↑
子域       二级域      顶级域(TLD)
```

| 层级 | 示例 | 说明 |
|------|------|------|
| **顶级域 (TLD)** | `.com`、`.cn`、`.edu`、`.org` | 最高层分类 |
| **二级域** | `baidu`、`csu` | 组织/个人注册的名字 |
| **子域** | `www`、`mail`、`api` | 二级域下的分区 |

常见顶级域含义：
- `.com` — 商业组织
- `.edu` — 教育机构（如 `csu.edu.cn` 就是中南大学）
- `.cn` — 中国国家域名
- `.org` — 非营利组织

## 5.3 DNS 解析过程

当你在浏览器输入 `www.baidu.com` 时，发生了以下过程：

```
你的电脑                              互联网
   │                                    │
   │  1. "www.baidu.com 的 IP 是啥？"    │
   │ ─────────────────────────────────► │  本地 DNS 服务器
   │                                    │      │
   │                                    │      │ 2. 问根DNS → 问.com DNS → 问 baidu.com DNS
   │                                    │      │
   │  3. "IP 是 220.181.38.148"          │      │
   │ ◄───────────────────────────────── │ ◄────┘
   │                                    │
   │  4. 用这个 IP 去访问百度服务器        │
   │ ─────────────────────────────────► │
```

简单来说：
1. 你的电脑先问本地 DNS 服务器："这个域名的 IP 是多少？"
2. DNS 服务器层层查询，最终返回对应的 IP 地址。
3. 拿到 IP 后，你的电脑就可以直接和目标服务器通信了。

::: info 动手试试
打开命令提示符/终端，输入：
```bash
nslookup www.baidu.com
```
你会看到百度域名对应的 IP 地址。每次运行结果可能不同，因为百度有很多服务器分布在不同地方。
:::

# 六、端口号 — 一台服务器上的"房间号"

## 6.1 为什么需要端口号？

一台服务器上可能同时运行着多个服务：网页服务、邮件服务、数据库服务……当数据到达服务器时，服务器怎么知道这个数据应该交给哪个程序处理？

答案就是 **端口号（Port）**。

> **类比：酒店房间号**
> 
> 快递送到"XX酒店"（IP 地址），但酒店有 100 个房间（端口）。快递员需要知道房间号才能把包裹送到正确的人手上。

## 6.2 常见端口号

| 端口号 | 对应服务 | 说明 |
|--------|---------|------|
| **80** | HTTP | 网页浏览（不加密） |
| **443** | HTTPS | 安全网页浏览（加密） |
| **22** | SSH | 远程登录 Linux 服务器 |
| **3306** | MySQL | 数据库 |
| **27017** | MongoDB | 数据库 |
| **3000** | 开发服务器 | Express 默认端口 |
| **5000** | 开发服务器 | Flask 默认端口 |
| **8080** | 代理/备用 HTTP | 常用于开发 |

::: warning 注意
端口号的范围是 0~65535。其中 0~1023 是"知名端口"，通常被系统服务占用。你自己开发时，建议使用 1024 以上的端口。
:::

## 6.3 IP + 端口 = 唯一服务

一个完整的网络地址需要 **IP + 端口**：

```
192.168.1.105:3000
    ↑           ↑
  IP 地址     端口号
（找到哪台电脑）（找到哪个程序）
```

当你运行 Flask 并看到 `http://127.0.0.1:5000` 时：
- `127.0.0.1` 是一个特殊 IP，表示"我自己"（本机回环地址）。
- `5000` 是 Flask 监听的端口号。

# 七、URL 的完整结构

你在浏览器地址栏输入的那串文字叫做 **URL（统一资源定位符）**。让我们来拆解它：

```
https://www.example.com:443/path/to/page?name=value&key=abc#section
  ↑        ↑            ↑       ↑              ↑              ↑
协议      域名          端口    路径           查询参数        锚点
```

| 部分 | 示例 | 说明 |
|------|------|------|
| **协议** | `https://` | 用什么规则通信（HTTP/HTTPS/FTP） |
| **域名** | `www.example.com` | 目标服务器的"名字" |
| **端口** | `:443` | 可省略。HTTP 默认 80，HTTPS 默认 443 |
| **路径** | `/path/to/page` | 服务器上的具体资源位置 |
| **查询参数** | `?name=value&key=abc` | 给服务器的附加信息 |
| **锚点** | `#section` | 页面内跳转位置，不会发给服务器 |

::: tip 省略规则
- 端口可以省略：`https://www.baidu.com` 等同于 `https://www.baidu.com:443`
- 路径可以省略：省略时默认为 `/`
- 参数和锚点是可选的
:::

# 八、客户端与服务器

## 8.1 谁是客户端？谁是服务器？

在网络通信中，角色非常明确：

| 角色 | 定义 | 例子 |
|------|------|------|
| **客户端（Client）** | 发起请求的一方 | 你的浏览器、手机 App |
| **服务器（Server）** | 接收请求并响应的一方 | 百度的服务器、你的 Flask 应用 |

> **类比：点外卖**
> 
> 你是客户端（下单的人），餐厅是服务器（做菜的人）。你发出请求（点单），餐厅返回响应（送餐）。

## 8.2 你的电脑可以同时是两者

这一点很重要：**客户端和服务器不是固定的设备，而是临时的角色**。

- 当你打开浏览器访问百度时，你的电脑是 **客户端**。
- 当你运行 `python app.py` 启动 Flask 服务时，你的电脑是 **服务器**。
- 当你用浏览器访问自己电脑上的 Flask 服务时，你的电脑**同时**扮演两个角色。

```
┌─────────────────────────┐
│        你的电脑           │
│                          │
│  浏览器 ──────► Flask     │
│ (客户端)      (服务器)    │
│                          │
│  127.0.0.1:5000          │
└─────────────────────────┘
```

这就是为什么开发时访问 `http://127.0.0.1:5000` 或 `http://localhost:5000` 能工作——你是在自己电脑上同时运行客户端和服务器。

::: info localhost 是什么？
`localhost` 是 `127.0.0.1` 的别名，永远指向"本机"。它是一个特殊的回环地址，数据包离开电脑后立刻又回来，根本不会经过路由器或互联网。
:::

# 九、小结

让我们回顾一下本篇学到的核心概念：

```
你打开浏览器，输入 www.baidu.com
         │
         ▼
    DNS 把域名翻译成 IP 地址（如 220.181.38.148）
         │
         ▼
    你的电脑（客户端）通过 IP 地址找到百度服务器
         │
         ▼
    通过端口号（80 或 443）找到服务器上的网页服务
         │
         ▼
    服务器返回网页数据，浏览器渲染后显示给你
```

| 概念 | 一句话解释 |
|------|-----------|
| **互联网** | 连接全球设备的巨型网络 |
| **IP 地址** | 设备在网络中的"门牌号" |
| **MAC 地址** | 设备硬件的"身份证号" |
| **域名** | IP 地址的"好记版本" |
| **DNS** | 把域名翻译成 IP 地址的"电话簿" |
| **端口号** | 一台服务器上不同服务的"房间号" |
| **URL** | 你在浏览器输入的完整地址 |
| **客户端/服务器** | 发请求的叫客户端，响应请求的叫服务器 |
