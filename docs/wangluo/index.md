## 💡 给大一新生的学习建议

后端开发不仅仅是写代码，更是**理解规则和流程**的过程。

1. **不要死记硬背**：你不需要背下所有的 HTTP 状态码或 Flask 的语法。学会**查文档**和**看报错**才是最重要的。
2. **多用调试工具**：多用 `print()` 或 `console.log()` 打印变量，多用 Apifox 观察接口返回。
3. **理解数据流**：当你写代码时，脑子里要有一张图：数据是怎么从浏览器传到你的函数，又是怎么存进数据库的。
4. **保持好奇心**：当你看到 `200 OK` 变成 `500 Internal Server Error` 时，那是你进步最快的时候。

## 🏗️ 后端核心思维（预习）

在开始具体技术学习前，请先在大脑中建立这两个概念：

### 1. 架构分层
后端代码通常遵循“向内依赖”的四层模型：
- **Router 层**: 负责接待请求。
- **Service 层**: 处理核心业务逻辑。
- **Repository 层**: 负责与数据库打交道。
- **Domain 层**: 定义最纯粹的数据实体。

### 2. 请求的生命周期
一次 HTTP 请求从发出到返回，会经历：
`浏览器` → `网关(Nginx)` → `Web容器` → `鉴权路由` → `业务处理` → `数据持久化` → `响应返回`。
